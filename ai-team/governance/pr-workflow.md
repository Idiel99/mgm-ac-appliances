# PR Workflow

## Rules
1. No direct pushes to main.
2. One story maps to one branch.
3. Every PR maps to one story.
4. Developer never merges the PR.
5. QA must provide evidence before lead review.
6. Eng Lead must review before user verification.
7. The human approves the PR merge.

## Branch naming
```text
story/STORY-###-short-description
```
